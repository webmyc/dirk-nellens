01 — OverviewWhat we're building

A fully interactive Human Design Bodygraph generator, embedded as a page or modal on dirk.respira.cafe. A visitor enters their birth data, gets a stunning visual of their unique energetic blueprint, downloads a branded PDF — and is invited to book a session with Dirk to understand what it means.

This is the site's primary lead generation tool. It should feel as refined as the rest of the site: unhurried, professional, deeply considered. Not a widget — an experience.

The strategic arc: curiosity → engagement → email capture → chart reveal → desire → booking. Every step of the design serves this arc without ever feeling like a funnel.

Success metrics this serves

  • Email list growth — every chart requires an email address
  • Session bookings — every chart ends with an invitation to work with Dirk
  • Brand authority — a beautiful, accurate chart positions Dirk at the same level as the best in the field
  • Shareability — the downloadable PDF carries Dirk's logo wherever it travels

Route

The chart generator lives at /chart. It is also accessible via a prominent CTA on the homepage and the sessions page. On desktop it can optionally open as a full-screen overlay from anywhere on the site.

02 — Technical ContextHuman Design primer for engineers

Before coding anything, the team needs to understand what a Human Design chart actually contains. Here is the minimum viable understanding.

The calculation

A Human Design chart requires two astronomical calculations, both using the Swiss Ephemeris — the same library used by professional astrology software:

  1. Personality (Conscious / Black): Planetary positions at the exact moment of birth.
  2. Design (Unconscious / Red): Planetary positions approximately 88 days (~88 degrees solar arc) before birth. This represents the "soul's imprint" before incarnation.

13 planetary bodies are used: Sun, Earth, Moon, North Node, South Node, Mercury, Venus, Mars, Jupiter, Saturn, Uranus, Neptune, Pluto.

Each planet's position maps to one of 64 Gates (corresponding to the 64 hexagrams of the I Ching). This mapping uses the Rave Mandala — a 360° wheel where each gate occupies a specific arc.

The 9 Centers

The Bodygraph has 9 energy centers, arranged in a specific geometric pattern representing the human body:

Ajna (Mind)

Triangle △. Awareness center. Mental certainty. Gates: 47, 24, 4, 17, 43, 11.

Throat

G / Identity

Diamond ◇. Love, direction, self. Gates: 1, 13, 25, 46, 2, 15, 10, 7.

Heart / Will (Ego)

Small triangle ▽. Willpower, ego. Gates: 21, 40, 26, 51.

Sacral

Square □ (large). Life force, sexuality, work. Gates: 5, 14, 29, 59, 9, 3, 42, 27, 34.

Solar Plexus

Triangle △. Emotions, desire, spirit. Gates: 6, 37, 22, 36, 30, 55, 49.

Spleen

Triangle ▽. Intuition, health, time. Gates: 48, 57, 44, 50, 32, 28, 18.

Root

Square □ at bottom. Pressure, adrenaline. Gates: 58, 38, 54, 53, 60, 52, 19, 39, 41.

Centers: Defined vs Undefined vs Open

  • Defined (colored): When both gates forming a channel are activated by either Personality or Design calculation. This center runs consistently. Color = terracotta for Design, black for Personality, striped for both.
  • Undefined (white with gate activations): Has some gate activations but no complete channel.
  • Open (completely white): No gate activations at all. Maximum receptivity.

The 36 Channels

Channels are the connections between centers. A channel is defined when both of its two gate endpoints are activated. When a channel is defined, it defines both connected centers.

Profile — the 12 archetypes

Profile is derived from the Lines of the Sun gate in both Personality and Design calculations. Each of the 64 gates has 6 lines, so a gate reading like "32.5" means Gate 32, Line 5. The profile combines Personality line + Design line (e.g., "1/3 Investigator/Martyr").

Authority — decision-making style

Authority is determined by which motor centers are defined and in what order:

  1. Emotional Solar Plexus — Solar Plexus defined. Must wait for emotional clarity.
  2. Sacral — Sacral defined, Solar Plexus not. Gut response in the moment.
  3. Splenic — Spleen defined, Sacral & Solar Plexus not. In-the-moment intuition.
  4. Ego / Heart Projected — Heart defined, no Sacral/Solar Plexus/Spleen. Willpower-based.
  5. Self-Projected (G Center) — G Center connected to Throat. Talking it out.
  6. Mental / None (Sounding Board) — No motors defined. Projector-specific. Outer authority.
  7. Lunar — Reflectors only. Waiting a full lunar cycle.

Incarnation Cross

Derived from the 4 gate numbers of Sun and Earth in both Personality and Design. There are 192 crosses in 4 families: Right Angle, Left Angle, Juxtaposition, Quarter. Display the name (e.g., "Right Angle Cross of the Unexpected") and the 4 gate numbers.

03 — UX ArchitectureUser flow — all 7 steps

1

Entry — The Intake Form

User lands on /chart or opens the overlay from any page. They see a clean, editorial form: full name, date of birth, time of birth, location of birth. A brief, warm explanation of why each field matters. No technical jargon.

2

Loading — The Wait

After form submission, an animated loading state plays for 2–4 seconds. Not a spinner — a breathing, animated version of the bodygraph outline, slowly filling in. This builds anticipation.

3

Preview — Chart Silhouette

The bodygraph appears, but blurred (filter: blur(8px)) with a translucent overlay. The user can see their chart taking shape — colors, defined centers — but cannot read the details. Type and Profile appear clearly above. This creates desire.

4

Email Gate — Unlock Full Chart

A centered overlay card asks for an email address to "reveal your full chart and receive a downloadable PDF." One field. One button. The email is used for the PDF and added to Dirk's list. After entering email, chart unblurs with a smooth animation.

5

Full Chart Reveal

The complete bodygraph renders in full fidelity. Below: a data panel explaining the user's Type, Strategy, Authority, Profile, defined channels, and defined centers. The language is warm and accessible, not clinical.

6

PDF Download

A "Download Your Chart" button generates a beautifully designed PDF on the fly. The PDF includes the bodygraph image, all chart data, Dirk's logo, a short description of each defined center, and an invitation to book a session. Requires the email address (already captured).

7

Reading Invitation

After the chart fully reveals, a section below (or a delayed modal after 15 seconds) invites the user to book a session with Dirk. The copy speaks directly to what they just discovered about themselves. This is the conversion moment.

Accurate Human Design calculation requires Swiss Ephemeris planetary position data. There are three valid approaches. Approach A is strongly recommended.

Approach B — Open-source Swiss Ephemeris (self-hosted)

Use swisseph npm package (Node.js binding to the Swiss Ephemeris C library). More complex to implement but fully self-contained. Requires implementing the full gate-mapping algorithm from planetary degrees to I Ching hexagrams.

// Install
npm install swisseph

// Calculate planetary positions
const swe = require('swisseph');

// Julian day number from birth date/time
const jd = swe.julday(year, month, day, hourDecimal, swe.GREG_CAL);

// Get Sun position for Personality
const sunPos = swe.calc_ut(jd, swe.SE_SUN, swe.FLG_SWIEPH);
// sunPos.longitude = degrees (0-360)

// 88 degrees before = Design date
// Calculate: find JD where Sun was 88° earlier
// Then recalculate all 13 planets for that JD

// Map degree → Gate using Rave Mandala
// The 64 gates occupy specific arcs on the 360° wheel
// Gate sequence (not sequential): 41,19,13,49,30,55,37,63,22,36,25,17,21,51,42,3,27,24,2,23,8,20,16,35,45,12,15,52,39,53,62,56,31,33,7,4,29,59,40,64,47,6,46,18,48,57,32,50,28,44,1,43,14,34,9,5,26,11,10,58,38,54,61,60

function degreeToGate(degrees) {
  const GATE_ORDER = [41,19,13,49,30,55,37,63,22,36,25,17,21,51,42,3,27,24,2,23,8,20,16,35,45,12,15,52,39,53,62,56,31,33,7,4,29,59,40,64,47,6,46,18,48,57,32,50,28,44,1,43,14,34,9,5,26,11,10,58,38,54,61,60];
  const gateArc = 360 / 64; // 5.625° per gate
  const lineArc = gateArc / 6;  // 0.9375° per line
  const index = Math.floor(degrees / gateArc) % 64;
  const gate = GATE_ORDER[index];
  const lineOffset = (degrees % gateArc) / lineArc;
  const line = Math.floor(lineOffset) + 1;
  return { gate, line };
}

Approach C — External embed fallback

As a temporary fallback during development, embed a third-party chart generator (e.g., mybodygraph.com embed or similar) in an iframe, styled to match the site. This is not the final solution but allows testing the UX flow before the calculation engine is complete.

Channel lines

Each channel is a <line> or <path> drawn between the connecting points of two centers. The connecting points are the midpoints of the relevant edge of each center shape (e.g., the bottom midpoint of the Throat center connects to the top midpoint of the G Center).

Channel stroke width: 12px. This makes them visually prominent at all sizes.

Color adaptation from standard HD: Traditional Human Design uses red for Design and black for Personality. We're mapping those to the site palette: terracotta (#C8643C) replaces red, and deep brown-black (#2A2218) replaces black. The visual language is preserved, the palette is Dirk's.

Gate numbers

Small numbers appear along the edges of each center, showing which gates are active. Activated gates are bold; inactive (open) gates are shown as very light gray numbers. Gate number font: 10px, font-family matching site serif, positioned along the perimeter of each center shape.

Only show gate numbers for activated gates and their inactive pair (to show incomplete channels). The numbers are interactive on hover — hovering a gate number shows a tooltip with the gate's name.

Responsive behavior

  • Desktop (>900px): Bodygraph SVG centered at 440px wide. Planet columns visible on both sides.
  • Tablet (600–900px): Bodygraph full width minus padding. Planet columns hidden, shown in a tab below the chart.
  • Mobile (<600px): Bodygraph full width. Planet data in a collapsible accordion below. Chart at minimum 320px wide to remain legible.

Interactivity

  • Hover on any center: Tooltip appears with center name, function, and whether it's defined or open in this chart. For defined centers: include the keyword (e.g., "Consistent life force and sexuality").
  • Hover on any channel: Tooltip shows channel name (e.g., "Channel of Structuring") and whether it's conscious, unconscious, or both.
  • Hover on a gate number: Tooltip shows gate name (e.g., "Gate 43: Insight") and which planet activated it.

09 — Step 6PDF generation — full specification

The downloadable PDF is Dirk's most distributed brand asset after this feature launches. It needs to be as beautiful as a book's inside. Every person who generates a chart will likely share this PDF or return to it. The Dirk logo on the back page travels everywhere it goes.

PDF dimensions and format

A4 portrait (210×297mm). Generated server-side using puppeteer or @react-pdf/renderer. Two approaches — choose based on stack:

  • Puppeteer approach: Render a hidden HTML page that matches the PDF design exactly, then take a PDF screenshot with Puppeteer. Allows pixel-perfect CSS rendering including the SVG.
  • react-pdf approach: Declarative PDF layout in React. Less flexible but no headless browser needed. May require the bodygraph SVG to be converted to an image first.

Recommended: Puppeteer. The bodygraph SVG will render perfectly. The PDF will match the website design exactly. Host on Vercel with a dedicated serverless function or use a service like HTML/CSS to PDF API (pdfshift.io, htmlcsstoapi.com).

PDF generation API route

// /api/generate-pdf
import puppeteer from 'puppeteer';

export async function POST(req) {
  const { chartData, userData } = await req.json();

  // Verify email was captured before generating PDF
  if (!userData.emailVerified) {
    return Response.json({ error: 'Email required' }, { status: 401 });
  }

  const browser = await puppeteer.launch({ headless: true });
  const page = await browser.newPage();

  // Render the hidden PDF template route
  await page.goto(
    `${process.env.NEXT_PUBLIC_URL}/chart/pdf-template?data=${encode(chartData)}`,
    { waitUntil: 'networkidle0' }
  );

  const pdf = await page.pdf({
    format: 'A4',
    printBackground: true,
    margin: { top: '20mm', right: '20mm', bottom: '20mm', left: '20mm' },
  });

  await browser.close();

  return new Response(pdf, {
    headers: {
      'Content-Type': 'application/pdf',
      'Content-Disposition': `attachment; filename="HD-Chart-${userData.name}.pdf"`,
    },
  });
}

Accessibility

  • The bodygraph SVG: role="img" with a descriptive aria-label summarizing the chart (e.g., "Human Design Bodygraph for [Name]: Generator type, Profile 3/5, Sacral Authority. 4 defined centers: Ajna, Throat, G Center, Sacral.").
  • All interactive elements: keyboard navigable with visible focus states.
  • Tooltips: announced by screen readers via aria-live="polite".
  • Form: all fields have visible labels above (not just placeholders).
  • Color alone never conveys meaning — all states have text labels in addition to color coding.

Performance

  • The chart component is lazy-loaded (dynamic(() => import('./components/BodyGraph'), { ssr: false })). SVG is generated client-side after data returns.
  • PDF generation is triggered on demand only — no pre-generation.
  • Google Places proxy caches results for 5 minutes server-side to avoid rate limits.
  • Chart calculation result cached in sessionStorage keyed by birth data hash — if user refreshes, chart doesn't need to be recalculated.
